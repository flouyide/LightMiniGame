# 遗物系统 · 策划配置指南

> 给**策划**看的手册：不碰代码，只讲「在编辑器里点什么、填什么、怎么验证」。
> 相关：牌库配置见 `牌库系统说明.md`，商店卖哪些遗物见 `商店系统说明.md`。

---

## 0. 一句话理解遗物

遗物也和卡牌一样**按角色隔离**——每个角色有**自己独立的遗物库**。但遗物比卡牌简单：
**遗物没有「实例层」**，一件遗物就是一张模板（`RelicData`），全游戏唯一；谁拥有它，就直接记在某角色的遗物库里。

一个关键设计：**不同角色可以拥有 / 购买同一件遗物**。配置上靠「两个角色的遗物池里放同一件 `RelicData`」实现——所以你完全可以让女剑士和法师都带同一个「牛」遗物。

---

## 1. 策划配置总清单

- [ ] **① 建遗物模板**（`RelicData`）——定义每件遗物是什么
- [ ] **② 建总遗物库并分池**（`MasterRelicLibrary` → `pools`）——**最重要的一步**
- [ ] **③ 挂到商店**（`ShopManager.masterRelicLibrary`）
- [ ] **④ 角色初始遗物**（`CharacterData.startingRelics`）

---

## 2. ① 建遗物模板（RelicData）

**操作**：Project 右键 → `Create → CardGame → Relic Data`，重命名为遗物名。

| 字段 | 策划要填什么 | 说明 / 例子 |
| --- | --- | --- |
| `relicId` | 稳定唯一 ID | 英文/数字，**不重复、别改**（持久化/查重用）。例：`relic_ox` |
| `relicName` | 显示名 | 「牛」 |
| `description` | 描述 | 遗物效果说明，显示在清单里 |
| `icon` | 图标（Sprite） | 商店/清单里显示的遗物图标 |
| `value` | 商店售价（金币） | 如 `50`。商店卖这件遗物标这个价 |
| `grade` | 品级 | 下拉选品级，清单里显示用 |
| `sortOrder` | 排序权重 | 可选；数字小排前面 |

> 图标资源：项目里已有 `Assets/Sprites/Relics/` 下的图（牛/船锚/面包/烛台/火龙果/芒果等），直接拖进 `icon` 即可。

---

## 3. ② 建总遗物库并分池（最关键）

遗物**不是**散着放的——它们统一收在「总遗物库」里，并且**按角色分成不同的池（pool）**，每个角色一个池。商店抽取、清单显示都从池里来。

**A. 建总遗物库资产**
- Project 右键 → `Create → CardGame → Master Relic Library`。
- （项目里一般已有 `Assets/ScriptableObjects/MasterRelicLibrary.asset`，直接双击打开即可，无需新建。）

**B. 分池（每个角色加一项）**
在 Inspector 找到 **`Pools`** 列表，点右下角 `+` 一次 = 新增一个 `CharacterRelicPool`（一个角色的遗物池）：

| 字段 | 策划要填什么 |
| --- | --- |
| `Character` | 拖入**对应角色**的 `CharacterData` |
| `Relics` | 展开它，把 `Size` 设为该角色遗物数量，再把每件 `RelicData` 逐个拖进 `Element 0 / 1 / 2…` |

**每个角色重复一次**（角色1 加一项、角色2 再加一项）。

> **顺序即身份**：`Pools` 列表里 **第 0 项 = 角色1，第 1 项 = 角色2**，和 `GameConfig.characters` 的顺序一致，也和商店 `character1RelicRatio` 的「角色1」对应。

### 想让两个角色能买同一件遗物？
把**同一件** `RelicData` 同时拖进两个角色的 `Relics` 列表（比如角色1 和角色2 的池里都放「牛」）。
这样商店抽取时两角色都能抽到、买到它，互不干扰。

---

## 4. ③ 挂到商店

把 `MasterRelicLibrary` 资产拖到场景里 `ShopManager` 组件的 **`masterRelicLibrary`** 字段。
（如果 `ShopManager` 上已经有其它引用，只是补这一个即可。）

> 商店抽取遗物的来源就是这个 `pools`。**只要 `Pools` 不为空，旧的全局列表就不再生效**——所以务必把遗物都分到各角色池里，别留空。

---

## 5. ④ 角色初始遗物（开局就有）

打开各 `CharacterData` 资产，找到 **`startingRelics`** 字段，把该角色开局就拥有的遗物（`RelicData`）拖进去。

- 这是「开局自带」，和商店买到的是两回事，但都进同一个角色的遗物库。
- 想让两个角色开局都带某遗物？同一件 `RelicData` 分别拖进两个角色的 `startingRelics` 即可。

---

## 6. 常见坑 & 排错

| 现象 | 原因 / 解决办法 |
| --- | --- |
| 商店里一件遗物都刷不出来 | `MasterRelicLibrary.pools` 是空的（没分池），或没挂到 `ShopManager.masterRelicLibrary` |
| 商店只刷出某几件、其它抽不到 | 那些没被放进任何角色的 `Relics` 列表；检查分池是否完整 |
| 两角色想买相同遗物却做不到 | 该遗物只在一个角色的池里；把它**也**拖进另一个角色的 `Relics` |
| 清单界面角色切换按钮不对 | `RelicInventoryPanel.prefab` 内 `Character1/Character2` 按钮需已绑定到 `RelicInventoryPanelUI` 的 `CharacterButton1/2`（一般已配好，无需动） |
| `relicId` 冲突 / 存档错乱 | `relicId` 是钥匙，别重复、别改 |

---

## 7. 验证清单（进 Unity 后做一遍）

- [ ] 总遗物库 `pools` 里每个角色都至少有一项，且 `Relics` 已填遗物。
- [ ] 商店进店能刷出遗物，且遗物上方显示所属**角色名**（金色）。
- [ ] 把同一件遗物放进两角色的池，进店确认两角色都能抽到/买到它。
- [ ] 清单界面（遗物按钮）能切换角色，显示「角色名 (数量)」。
- [ ] 购买遗物后进入**对应角色**遗物库。

> 定制引擎无法命令行编译；配置后请在 Unity 打开确认无报错并实机走一遍。
