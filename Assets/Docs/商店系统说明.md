# 商店系统 · 策划配置指南

> 给**策划**看的手册：不碰代码，只讲「在编辑器里点什么、填什么、怎么调数值、怎么验证」。
> 前提：卡牌/遗物已经在「牌库系统」「遗物系统」里配好（见对应文档）。商店只是**把它们按规则摆出来卖**。

---

## 0. 一句话理解商店

商店每次开门，会从「总牌库」「总遗物库」里**按角色比例**随机抽一些卡和遗物摆上架。玩家用金币买，买到的卡/遗物进**对应角色**的库。还能花金币**删牌**。

- 卡和遗物都带「属于哪个角色」（角色1 / 角色2），买错人不会进错库。
- 抽卡和抽遗物**规则不同**（见第 6、7 节），策划要心里有数。

## 1. 策划配置总清单

- [ ] **① 建商店配置**（`ShopConfig`）——所有可调数值都在这里
- [ ] **② 挂到 ShopManager**——把配置和「货源地」拖进组件
- [ ] **③ 挂 ShopPanelUI**——卡面预制体 + 删牌复用的牌库面板
- [ ] **④ 调数值 & 验证**

## 2. ① 建商店配置（ShopConfig）—— 核心参数表

**操作**：Project 右键 → `Create → CardGame → Shop Config`，重命名为 `ShopConfig`（项目里一般已有 `Assets/ScriptableObjects/ShopConfig.asset`，直接打开改即可）。

| 字段 | 默认 | 策划怎么设 | 说明 / 例子 |
| --- | --- | --- | --- |
| `shopCardCount` | 5 | 一次店里摆几张卡 | 设 `8` 则货更全；空间够就调大 |
| `shopRelicCount` | 3 | 一次店里摆几件遗物 | 设 `4` 等多一点 |
| `character1CardRatio` | 0.5 | 卡牌中**角色1**占的比例 | `0.6` = 60% 卡来自角色1池，角色2 拿剩下 40% |
| `character2CardRatio` | 0.5 | **仅参考显示**，实际由上面的推导 | 不用改，程序按总数−角色1数量算 |
| `character1RelicRatio` | 0.5 | 遗物中**角色1**占的比例 | 同卡牌逻辑；`0.5` = 两角色各一半 |
| `character2RelicRatio` | 0.5 | **仅参考显示** | 同上 |
| `removalCount` | 3 | 每次进店可删几次牌 | 想让玩家多删就调大 |
| `removalBasePrice` | 75 | 删第一张牌的基础价 | 金币数 |
| `removalPriceStep` | 25 | 每多删一张涨价多少 | 见第 8 节涨价公式 |

> **比例怎么理解**：`character1CardRatio` 决定「店里卡牌里有多少来自角色1的池」，角色2 自动拿剩下的。两个 `Ratio` 加起来不必等于 1，程序用「总数 − 角色1数量」算角色2。

## 3. ② 挂到 ShopManager（Inspector 拖拽）

场景里放一个 `ShopManager`（或已存在），在 Inspector 把以下资产拖进去：

| 字段 | 拖什么 |
| --- | --- |
| `shopConfig` | 上面的 `ShopConfig` 资产 |
| `masterCardLibrary` | `MasterCardLibrary` 资产（总牌库，见牌库文档） |
| `masterRelicLibrary` | `MasterRelicLibrary` 资产（总遗物库，见遗物文档） |

> 这三者缺一不可，否则商店抽不到货。

---

## 4. ③ 挂 ShopPanelUI（卡面 & 删牌面板）

场景里商店面板上的 `ShopPanelUI` 组件，需要绑：

| 字段 | 拖什么 | 说明 |
| --- | --- | --- |
| `attackCardPrefab` | `攻击牌.prefab` | 卡牌按 `cardType` 选预制体画 |
| `armorCardPrefab` | `护甲牌.prefab` | 同上 |
| `buffCardPrefab` | `增益牌.prefab` | 同上 |
| `cardLibraryPanel` | `CardLibraryPanel` 实例 | 删牌时复用这个面板当选择界面 |

> 删牌按钮（`Service`）一般 prefab 里已有，无需额外拖；若 `ShopPanelUI` 有 `serviceButton` 字段，可留空（程序会自动找 prefab 里的 `Service`）。

## 5. 卖哪些货、由谁决定？

商店**不会**凭空决定卖什么卡/遗物——货全部来自「总牌库 / 总遗物库」的分池：

- 卡牌来源 = `MasterCardLibrary.pools`（每个角色一组 `CardData`）→ 见牌库文档步骤 ②/③/④。
- 遗物来源 = `MasterRelicLibrary.pools`（每个角色一组 `RelicData`）→ 见遗物文档步骤 ②。

**策划要做的**：把想卖的卡/遗物，分别放进对应角色的 `pools`。没放进池的，商店永远刷不出来。

---

## 6. 抽卡规则（和遗物不一样！）

卡牌抽取用**「跨角色共享排除」**：

- 先抽角色1 的池，抽到的卡模板记下来；再抽角色2 时**把角色1 抽到的排除掉**。
- 结果：**同一家店，一个角色抽到某张卡模板，另一个角色不会再抽到同一模板**（避免重复摆同一张）。

> 当前卡牌抽取**不检查「玩家已拥有」**——玩家手里已有的卡，店里仍可能再刷出来（重复购买由玩家自己决定）。如果你希望「已拥有的卡不再出现在店里」，需要程序再改一版（当前未做）。

---

## 7. 抽遗物规则（和卡牌不一样！）

遗物抽取是**「按角色独立」**：

- 每个角色**单独**抽自己的池，**只排除「自己」已拥有的遗物**，不碰别的角色。
- 结果：**不同角色可以抽到 / 购买同一件遗物**（比如两角色池里都放了「牛」，两家店都能买到它），且一个角色的抽取不影响另一个。

> 这正是遗物系统允许「两角色共享同一遗物」的原因。配置方法：把同一件 `RelicData` 拖进两个角色的 `pools`（见遗物文档）。

---

## 8. 删牌费用怎么算（策划最常被问）

公式：

```
本次删牌价 = removalBasePrice + removalPriceStep × 全局已删牌次数
本店还能删几次 = removalCount − 本店已删次数
```

关键点（容易误解）：

- **本店次数**（`removalCount`）：每次进新店**重置**为满额。比如设 3，每家店都能删 3 张。
- **全局涨价**（`removalPriceStep`）：删牌价格**跨店一直涨**，永不清零。第一家店删第 1、2、3 张分别是 `75 / 100 / 125`；第二家店**起始价就是 `150`**（`75 + 25×3`），不会再回到 75。

示例（`base=75, step=25`）：
| 操作 | 累计全局删牌 | 本次价格 |
| --- | --- | --- |
| 店1 删第1张 | 1 | 75 |
| 店1 删第2张 | 2 | 100 |
| 店1 删第3张 | 3 | 125 |
| 店2 删第1张 | 4 | 150 |

> 想让删牌「便宜点、别涨太狠」：调小 `removalPriceStep`（如 10）或调大 `removalBasePrice` 起步门槛策略按设计来。

## 9. 常见坑 & 排错

| 现象 | 原因 / 解决办法 |
| --- | --- |
| 商店空空如也 | `ShopConfig` / 两个 `Master*Library` 没挂到 `ShopManager`；或总库 `pools` 是空的 |
| 卡牌/遗物只来自一个角色 | `character1CardRatio` / `character1RelicRatio` 设成了 1（或 0），调到 0.5 左右 |
| 想两角色都能买到某遗物却不行 | 该遗物只在一个角色的 `pools` 里；拖进两个角色的池 |
| 删牌按钮点了没反应 | 本店 `removalCount` 用完了；或 `cardLibraryPanel` 没绑 |
| 删牌价格一直不变 | 其实在涨，但你看的是「本店」；跨店才会明显上涨（全局累计） |
| 卡面显示「[缺少 X 预制体]」 | `ShopPanelUI` 的三个卡面预制体没拖（攻击/护甲/增益） |

- [ ] `ShopPanelUI` 三张卡面预制体 + `cardLibraryPanel` 已绑。
- [ ] 实机进店：卡牌按角色比例出现、遗物标了角色名、购买入库、删牌价格随删牌次数上涨。
