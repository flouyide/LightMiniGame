# 章节配置（ChapterConfig）· 策划配置指南

> 给**策划**看的手册：不碰代码，只讲「在编辑器里建什么、点什么、填什么、怎么调」。
> 这一份只管「一章长什么样、刷什么事件、背景怎么变」。事件池里每个事件的具体效果怎么配，见第 5 节速查，完整设计看 `特殊事件效果系统设计.md`。

---

## 0. 章节

每一章是一个 `ChapterConfig` 资产：它决定这章叫什么名字、玩家能翻几页、从哪一堆事件里随机抽页面、以及角色理智高低时背景图换不换。

- 游戏跑起来时，从 `GameConfig.asset` 的 `chapters` 列表读这一章。
- 每翻一页 = 从 `events` 池里随机抽 3 张事件卡给玩家选，选完扣一次「剩余选页数」。
- 剩余选页数归零，本章结束，进入下一章（或结算）。

---

## 1. 策划配置总清单

- [ ] **① 建一个 ChapterConfig 资产**
- [ ] **② 填「章节信息」三组字段**（名字 / 选页数 / 事件池）
- [ ] **③ 填「背景」两组图 + 理智阈值**
- [ ] **④ 往 events 池里拖事件**（每个是一条 `PageEventData`）
- [ ] **⑤ 把资产拖进 `GameConfig.asset` 的 `chapters`**（否则游戏读不到这一章）

---

## 2. ① 怎么建一个 ChapterConfig

Project 窗口里右键 → `Create → LightGame → Chapter Config`，重命名成这章的名字（比如 `Chapter2`）。建好后双击在 Inspector 里编辑。

> 项目里每章一个资产，互不共享。想新增一章就再建一个，别在同一个资产上改写。

---

## 3. ② 字段参数表（章节信息）

| 字段 | 默认 | 策划怎么设 | 说明 / 例子 |
| --- | --- | --- | --- |
| `chapterName` | 第一章 | 章节显示名 | 顶部栏会显示这个名字，纯文本，随便写，比如「霜降之卷」 |
| `maxSelections` | 10 | 这一章玩家能翻几页 | 设 `6` 章就短一点，设 `15` 章更长。每选 1 页减 1，减到 0 本章结束 |
| `events` | 空 | 拖进本章的事件池 | 见下面第 4 节 |

`maxSelections` 是这章的「总页数预算」。它跟事件池大小没关系：池子大只是每次抽到的不那么重复，但翻页次数只由这里定。

---

## 4. events 事件池怎么填

`events` 是一个列表，每格放一条 `PageEventData` 资产（事件本身的配置见特殊事件文档）。操作：

1. 先把每条事件按 `PageEventData` 配好（标题、描述、选项、每个选项挂的 `EffectData` 效果）。
2. 回到这个 ChapterConfig，把配好的 `PageEventData` 资产一条条拖进 `events` 列表。

刷新规律（程序行为，策划心里有数即可）：

- 每次翻页，从 `events` 里**随机抽 3 条**摆出来给玩家选。
- 池子越大，连续翻页时撞到重复事件的概率越低。池子太小（比如只有 3 条），翻几页就开始反复出现同样的事件。
- 池子是空的：本章刷不出页面。程序会找不到可显示的卡，表现为卡住或空白，所以至少放几条进去。

## 4.1 单条事件的刷新开关（isRepeatable / isFinalNode）

每个事件（`PageEventData`）身上有两个开关，控制它在章节里「什么时候刷、刷几次」。配事件时（第 4 节第 1 步）顺手设好。

### isRepeatable —— 能不能重复出现
- 不勾（默认）：这个事件一旦被选过（完成），本章不会再刷出来。适合只发生一次的事件，比如「遇到商人」「拿到某件关键物品」。
- 勾上：事件可以反复出现，哪怕已经完成过。适合通用填充事件，比如「路边营地休息」。
- 背后逻辑：程序按 `eventId` 记录已完成事件。所以每个事件必须填一个唯一的 `eventId`，否则两个事件会被当成同一个，重复控制会判断错。

### isFinalNode —— 是不是最终节点（Boss）
- 不勾（默认）：普通事件，正常参与随机。
- 勾上：标记为最终节点（Boss 关）。界面上这张卡会显示一个 Boss 标记。
- 出现时机：当本章剩余选页数归零时，如果最终节点还没完成，程序会强制让它出现（Boss 在章节末尾必到）。在那之前，它也能像普通事件一样正常随机刷出来。一旦完成过，就跳过不再出现。
- 注意：强制出现时只取候选列表里的第一个，所以一章通常只放一个最终节点。放多个的话，只有第一个会被强制在末尾出现。

### 一点提醒
这两个开关都依赖 `eventId`。没填或重复，重复/最终逻辑就会判断错。配事件时先给 `eventId` 起个唯一名，再动这两个勾。

## 5. 事件里的效果（EffectType）速查

每个事件（`PageEventData`）能挂一串效果。效果写在两处之一：

- Battle / Shop / Rest 类型的书页：写在 `defaultEffects`，点「进入」直接生效。
- Event 类型的书页：写在每个选项 `options[].effects` 里，玩家选了那个选项才生效。

一条效果 = 一个 `EffectData`，先选 `type`（效果种类），再按种类填对应字段。下面按 `type` 逐个说。

### GainItem / LoseItem —— 剧情物品（目前没用，如果以后有道具设计可以用上）
用途：纯叙事，告诉玩家获得了或失去了某物。项目目前没有物品背包系统，所以不会真的把东西放进库存，也不影响任何数值。
怎么配：`itemDesc` 写一句描述，比如「古老的钥匙」。
例子：选项「拿走信物」挂 `GainItem`，`itemDesc="褪色的护符"`。
注意：它只是文本提示，别指望触发后续逻辑。

### EnterBattle —— 进入战斗
用途：选了这个选项（或点了进入）就进入一场战斗。
怎么配：选 `EnterBattle` 即可，无额外参数。
注意：战斗系统接好前，这里只是个标记。

### ModifyAttribute —— 改玩家属性
这是统一处理「加/减 力量、敏捷、金币、生命、理智」的效果。配置分三步：
1. `attributeOp`：先选 Gain（加）或 Lose（减）。
2. `targetAttribute`：再选改哪个属性：Strength（力量）/ Agility（敏捷）/ Lifesteal（吸血）/ CritRate（暴击率）/ CritDamage（暴伤）/ Gold（金币）/ HP（生命）/ Sanity（理智）。
3. `amount`：填正数，表示增减的量。方向已由 `attributeOp` 定，别填负数。

例子：让玩家 +50 力量，就设 `ModifyAttribute` + `attributeOp=Gain` + `targetAttribute=Strength` + `amount=50`。
例子：扣 30 金币，就设 `ModifyAttribute` + `attributeOp=Lose` + `targetAttribute=Gold` + `amount=30`。
注意：力量/敏捷/吸血/暴击率/暴伤是跨战斗保留的（重开游戏才归零）；金币/生命/理智按局内数值走。生命不会超过上限，金币/理智不会低于 0。

### GrantRelic —— 发遗物给一个角色
用途：让某个随机角色获得一件遗物。
怎么配：拖具体遗物到 `relic` 字段就发这件；`relic` 留空则从该角色自己的遗物池里随机抽一件。
例子：`relic=牛`，某角色获得「牛」。
注意：发放对象是随机单个角色，不是全员。

### GrantCard —— 发卡牌给一个角色
用途：让某个随机角色获得指定卡牌。
怎么配：拖 `card`（具体卡），`amount` 填发几张。
例子：`card=火球术`，`amount=2`，某随机角色拿到 2 张火球术。
注意：同 GrantRelic，发放给随机单个角色。

### EnterDiscountShop —— 进入特价商店
用途：开一家打折的商店，玩家能低价买卡/遗物。
怎么配：`discountRatio` 填折扣比例，`0.6` = 6 折（售价 = 原价 × 比例）。范围 0.1 到 1，1 表示不打折。
例子：`discountRatio=0.6`，店里所有卡/遗物打 6 折，进店时界面顶部显示「特价商店（6折）」角标。
注意：每个特价店事件的折扣单独配，不同事件可以是不同折数。

### RemoveCard —— 删牌（要点选）
用途：让玩家从自己牌库里删掉一张卡（免费、单次）。
怎么配：选 `RemoveCard` 即可，无额外参数。
运行时：触发后弹出牌库面板，玩家点一张卡就删掉；点关闭则跳过，后续效果照常执行。
注意：这是交互式效果，会暂停效果序列等玩家操作完再继续。和商店里花金币删牌是两回事。

### AddKeywordToCard —— 给卡加词条（要点选）
用途：让玩家给某一张卡附加一个词条（比如「燃烧」「穿透」）。
怎么配：`keywordToAdd` 选要加的词条（`KeywordType` 位标枚举，可多选）。
运行时：弹出牌库面板，玩家点一张卡，词条按位追加到那张卡上（只改这一张卡，不影响其他副本）。
注意：也是交互式效果，点关闭则跳过。

### 几个通用字段
- `amount`：ModifyAttribute 的增减量、GrantCard 的发卡张数都用它。
- `itemDesc`：只给 GainItem / LoseItem 用，写描述文本。

### 一条选项挂多个效果
`effects` 是个列表，可以叠好几条。比如一个选项同时 +50 生命 且 -30 金币：加两条 `ModifyAttribute`（HP/Gain/50 和 Gold/Lose/30）。效果按列表顺序依次结算；交互式效果（删牌/加词条/特价店）会暂停等玩家操作完，再继续后面的。

## 6. ③ 背景按理智切换（三个字段）

角色有一个叫 `Sanity`（理智）的数值，会随事件涨跌。背景图根据它实时换：

| 字段 | 拖什么 | 说明 |
| --- | --- | --- |
| `NormalBG` | 一张 Sprite | 理智**高于或等于**阈值时显示的背景 |
| `AbnormalBG` | 一张 Sprite | 理智**低于**阈值时显示的背景 |
| `sanityThreshold` | 整数，默认 50 | 分界点：Sanity >= 50 用 NormalBG，< 50 用 AbnormalBG |

切换逻辑：每当场上理智变化，程序比较 `Sanity` 和 `sanityThreshold`，选对应的图。所以你要准备两张氛围不一样的图（比如正常图亮一点、异常图压抑一点），阈值控制从哪一刻切。

- 想让背景「更早」变异常：把 `sanityThreshold` 调高（比如 70），理智稍微掉一点就切。
- 想让背景「很难」变异常：把阈值调低（比如 20）。

> `NormalBG` 和 `AbnormalBG` 都留空也不会报错，但背景就是空的。至少填一张 `NormalBG` 保证有图可显示。

## 7. ⑤ 别忘了挂到 GameConfig

ChapterConfig 本身只是「一章的配方」。游戏实际读的是 `GameConfig.asset` 里的 `chapters` 列表：

1. 选中 `GameConfig.asset`，在 Inspector 找到 `chapters`（一个列表）。
2. 把刚建好的 `ChapterConfig` 资产拖进对应那一格（第几章就拖到第几位）。
3. 顺序错了，游戏会按列表顺序进章，所以位置要对。

漏掉这步是最常出的问题：资产配得再好，没进 `chapters` 列表，运行时这章根本不会被加载。

## 8. 常见坑 & 排错

| 现象 | 原因 / 解决办法 |
| --- | --- |
| 顶部栏章节名显示不对 | `chapterName` 没改，还是默认的「第一章」；或 GameConfig 里章节顺序拖错了 |
| 翻几页就开始反复出现同样的事件 | `events` 池太小，多拖几条 `PageEventData` 进去 |
| 翻页后卡住 / 没有页面可点 | `events` 是空的，至少放几条事件 |
| 背景一直不切换 | `sanityThreshold` 设太高（比如 100），理智永远够不到；或 `NormalBG`/`AbnormalBG` 只填了一张 |
| 背景是空白 | 两张背景图都没拖，至少填 `NormalBG` |
| 游戏里完全没这章 | `ChapterConfig` 没拖进 `GameConfig.chapters`；或拖错位置 |
| 章节一进来就直接结束 | `maxSelections` 设成了 0 或负数，调成正数 |

## 9. 验证清单（进 Unity 后做一遍）

- [ ] `ChapterConfig` 资产已建，名字改成这章的名字。
- [ ] `maxSelections` 设成了想要的页数（默认 10）。
- [ ] `events` 里拖进了至少几条配好的 `PageEventData`，且每条在特殊事件文档里能正常结算。
- [ ] `NormalBG` / `AbnormalBG` 都拖了图，`sanityThreshold` 按设计调好。
- [ ] 资产已拖进 `GameConfig.asset` 的 `chapters` 对应位置。
- [ ] 实机：顶部显示章节名；翻页扣剩余次数；事件池随机不重复；理智跨过阈值时背景图切换。

> 定制引擎无法命令行编译；配置后请在 Unity 打开确认无报错并实机验证。
