# 卡牌编辑器 · 策划配置指南

> 给**策划**看的手册：不碰代码，只讲「在编辑器里点什么、填什么、怎么验证」。
> 如果你要看旧的卡牌系统（CardData），请参考 `卡牌战斗系统_策划配置文档.md`。本文档以新的卡牌编辑器（CardEntry）为准。

---

## 0. 一句话理解卡牌编辑器

卡牌编辑器是 Unity 内的一个**工具窗口**，让你像填表格一样创建和编辑卡牌，不需要写任何代码。

打开方式：Unity 顶部菜单 → `Tools` → `卡牌编辑器` → `Card Editor`

界面分三栏：

```
┌─────────────┬──────────────────┬──────────────────┐
│  左栏        │  中栏             │  右栏             │
│  卡牌列表    │  基础信息编辑     │  效果编辑 / 预览   │
│  搜索/筛选   │  名称/费用/类型   │  效果列表/能力    │
│  新建/复制   │  描述/卡面/备注   │  预览/校验结果     │
│  删除        │  是否可升级       │                  │
└─────────────┴──────────────────┴──────────────────┘
```

## 1. 核心概念（先读这一节）

在动手之前，先理解几个关键词。这些词在编辑器里会反复出现。

### 1.1 卡牌 (CardEntry)

一张卡牌的完整定义。包含基础信息（名称、费用、类型等）和效果列表。卡牌保存为 `.asset` 文件，存在 `Assets/ScriptableObjects/CardEditor/` 目录下。

### 1.2 效果 (CardEffect)

打牌时**执行的一次性动作**。比如「造成 8 点伤害」「获得 5 点护甲」「抽 1 张牌」。

一张卡可以有多个效果，**按列表顺序依次执行**。比如：
1. 造成 8 点伤害
2. 获得 3 点力量
3. 抽 1 张牌

执行顺序就是 1 → 2 → 3，不会乱。

每个效果可以单独**启用/禁用**，方便调试。

### 1.3 能力 (AbilityData)

能力是**一种特殊的卡牌类型**。普通卡打出后执行效果就进弃牌堆了，但能力卡打出后会**一直挂在角色身上**，等到特定事件发生时自动触发效果。

比如「节拍器」：每回合第一次攻击时，自动让本次攻击必定暴击。

能力 = 触发时机 + 触发条件 + 触发后执行的效果列表。

能力触发后执行的东西**还是效果**，只是触发方式从「手动打牌」变成了「事件自动触发」。

### 1.4 基础版 vs 升级版

每张卡可以配置「基础效果」和「升级效果」两套完整的效果列表。

- 升级效果**不是**基础效果的补丁或差值，而是一套**独立、完整**的效果。
- 基础效果和升级效果的数量、类型、目标可以完全不同。
- 编辑器提供「从基础复制到升级」按钮，方便快速创建升级版的初始数据。

### 1.5 存在形式

决定卡牌打出后去哪里：

| 存在形式 | 意思 |
|---------|------|
| 普通 | 使用后进入弃牌堆（最常见） |
| 战斗内移除 | 使用后从本次战斗牌库移除（本战不再抽到） |
| 永久移除 | 使用后永久从玩家牌库移除 |

### 1.6 词条

特殊标记，目前支持：

| 词条 | 说明 |
|------|------|
| 无 | 没有特殊标记 |
| 回响 | 具体效果由战斗系统实现 |

> 注意：「消耗」不是词条。卡牌是否移除由「存在形式」字段决定。

## 2. 开始使用

### 2.1 打开编辑器

Unity 顶部菜单 → `Tools` → `卡牌编辑器` → `Card Editor`

### 2.2 创建卡牌数据库（首次使用）

如果是第一次使用，编辑器会提示「未找到 CardDatabase」。点击「创建 CardDatabase 资产」按钮即可。

数据库会创建在 `Assets/Resources/CardEditor/CardDatabase.asset`，用来管理所有卡牌。

### 2.3 快速生成示例卡牌

菜单 → `Tools` → `卡牌编辑器` → `生成示例卡牌`

这会自动生成 10 张示例卡牌（瞄准射击、校准、连锁爆发等），可以直接在编辑器中查看和学习。

---

## 3. 创建一张新卡牌

### 3.1 新建

点击左栏顶部的「新建卡牌」按钮。会生成一张空白卡，自动选中。

### 3.2 填写基础信息（中栏）

| 字段 | 说明 | 注意事项 |
|------|------|---------|
| 卡牌 ID | 唯一标识，不能为空，不能重复 | 建议用英文，如 `aim_shot` |
| 卡牌名称 | 显示给玩家看的名字 | 不能为空 |
| 卡面原画 | 拖入 Sprite 资源 | 可以后补，但校验会警告 |
| 品级 | 铜 / 银 / 金 | 影响商店价格和刷新概率（见第 8 节） |
| 卡牌类型 | 攻击 / 技能 / 能力 | 选「能力」时右栏会切换为能力编辑模式 |
| 存在形式 | 普通 / 战斗内移除 / 永久移除 | 见 1.5 节 |
| 基础费用 | 打出这张卡需要多少能量 | 不能小于 0 |
| 升级后费用 | 升级版的费用 | 勾选「升级」后才会出现 |
| 词条 | 无 / 回响 | 见 1.6 节 |
| 升级 | 勾选后可配置升级版 | 不勾选则只有基础版 |
| 基础描述 | 卡牌的描述文字 | 留空会自动生成效果描述 |
| 升级后描述 | 升级版的描述文字 | 同上 |
| 策划备注 | 给自己看的备注 | 不影响游戏 |

### 3.3 配置效果（右栏）

这是最核心的部分。下面详细讲。

## 4. 效果配置详解

### 4.1 添加效果

在右栏效果列表底部点击「+ 添加效果」按钮。

### 4.2 效果字段总览

每个效果有以下字段分组。**不是所有字段都会同时显示**——编辑器会根据效果类型自动只显示相关字段。

#### 基础设置

| 字段 | 说明 |
|------|------|
| 启用 | 关闭后此效果不执行（但不会被删除） |
| 效果名称 | 给自己看的名字，如「主伤害」「暴击追加」 |
| 效果类型 | 选择后下方自动切换对应字段（见 4.3） |
| 效果发起者 | 谁发起这个效果：玩家 / 选定敌人 / 随机敌人 / 所有敌人 / 自定义 |
| 效果目标 | 效果作用在谁身上：玩家 / 选定敌人 / 随机敌人 / 所有敌人 / 切换后角色 / 发起者自身 / 自定义 |

#### 数值表达式

不是只填一个数字，而是可以选择公式类型（见第 5 节详解）。

#### 次数

执行几次。比如攻击 3 次，就填 3。最少为 1。

#### 时机与持续

| 字段 | 说明 |
|------|------|
| 生效时机 | 立即 / 下回合开始 / 下次攻击 / 下一张牌 / 下次切换角色 / 每次切换角色 |
| 持续时间 | 一次行动 / 指定回合 / 战斗持续 / 本局永久 |
| 持续数值 | 当持续时间选了「指定回合」时，填回合数 |

#### 条件

效果是否执行，可以配置条件。详见第 6 节。

### 4.3 各效果类型的具体字段

选择不同的「效果类型」，下方显示的字段不同：

#### ① 伤害 (Damage)

| 字段 | 说明 |
|------|------|
| 伤害数值 | 数值表达式（见第 5 节） |
| 攻击次数 | 连击次数，最少 1。每次独立判定暴击 |
| 必定暴击 | 勾选后本次攻击一定暴击 |
| 无视护甲 | 勾选后伤害不减免护甲 |
| 施加破甲 | 勾选后同时施加破甲 |
| 破甲数值 | 施加多少点破甲 |

> **连击重要规则**：攻击次数大于 1 时，每一段都独立进行暴击判定、伤害计算和状态触发。不能只判定一次暴击后乘以总次数。

> **无视护甲**和**施加破甲**是两个独立的配置，互不影响。

#### ② 护甲 (Armor)

| 字段 | 说明 |
|------|------|
| 护甲数值 | 数值表达式（见第 5 节） |

> 护甲是否在回合结束时保留，**不由每张卡决定**，而是由玩家的全局战斗属性决定。所以这里没有「临时护甲/永久护甲」选项。

#### ③ 抽牌 (DrawCard)

| 字段 | 说明 |
|------|------|
| 抽牌数量 | 数值表达式 |
| 生效时机 | 立即 / 下回合开始 |

#### ④ 回费 (RestoreEnergy)

| 字段 | 说明 |
|------|------|
| 回复能量 | 数值表达式 |
| 生效时机 | 立即 / 下回合开始 |

#### ⑤ 修改属性 (ModifyAttribute)

增减益效果。可以修改玩家或敌人的各种属性。

| 字段 | 说明 |
|------|------|
| 增减益目标 | 玩家 / 敌人 / 当前手牌 / 下一张牌 / 指定类型牌 / 自定义 |
| 修改属性 | 力量 / 敏捷 / 暴击率 / 暴击伤害 / 破甲值 / 最大生命 / 当前生命 / 每回合抽牌数 / 每回合能量 / 流血值 / 货币 / 手牌费用 等 |
| 修改方式 | 增加 / 减少 / 乘算 / 覆盖 |
| 修改数值 | 数值表达式 |
| 持续时间 | 一次行动 / 指定回合 / 战斗持续 / 本局永久 |

#### ⑥ 修改手牌费用 (ModifyHandCost)

| 字段 | 说明 |
|------|------|
| 费用目标 | 下一张打出的牌 / 当前手牌全部 |
| 费用变化值 | 正数增加费用，负数降低费用 |

#### ⑦ 施加状态 (ApplyStatus)

| 字段 | 说明 |
|------|------|
| 状态类型 | 流血 / 破甲 / 力量 / 敏捷 / 疯狂 / 下次攻击增伤 / 下一张牌减费 / 暴击率提升 / 暴击伤害提升 |
| 层数 | 状态的层数或数值 |
| 可叠加 | 勾选后多次施加可叠加层数 |

#### ⑧ 自定义 (Custom)

| 字段 | 说明 |
|------|------|
| 效果脚本 | 拖入自定义效果脚本（ScriptableObject 资产） |
| 自定义参数 | 文本参数，由脚本读取 |

> 当效果太复杂、无法用通用字段表达时使用。详见第 7 节。

### 4.4 效果排序

每个效果右侧有 ↑ ↓ 按钮，可以调整效果在列表中的位置。**位置的先后就是结算的先后**。

### 4.5 复制效果

点击「复制」按钮，会在当前效果下方插入一份完全相同的副本。

### 4.6 删除效果

点击「删除」按钮直接移除。如果想临时停用但不删除，取消勾选「启用」即可。

## 5. 数值表达式

数值不只是填一个固定数字。点击公式类型的下拉框，有 4 种选择：

### 5.1 固定值

最简单的形式。直接填一个数字。

**例子**：伤害 = 8

### 5.2 基础值 + 属性

基础值加上某个属性的当前值。

**例子**：伤害 = 6 + 力量

如果当前力量是 3，则伤害 = 6 + 3 = 9。

### 5.3 基础值 + 属性 × 系数

基础值加上属性乘以一个系数。

**例子**：伤害 = 5 + 力量 × 0.5

如果当前力量是 4，则伤害 = 5 + 4 × 0.5 = 7。

### 5.4 基础值 + 计数 × 系数

基础值加上某个计数器乘以系数。和「属性加成」类似，但引用的是计数器而非属性。

### 5.5 可引用的属性 / 计数

| 名称 | 说明 |
|------|------|
| 力量 | 玩家力量值 |
| 敏捷 | 玩家敏捷值 |
| 当前生命 | 玩家当前 HP |
| 最大生命 | 玩家最大 HP |
| 已损失生命 | 最大生命 - 当前生命 |
| 当前理智 | 玩家当前理智值 |
| 本回合已损失理智 | 本回合内累计失去的理智 |
| 本场战斗累计失去理智 | 本场战斗中累计失去的理智 |
| 流血值 | 目标的流血层数 |
| 破甲值 | 目标的破甲层数 |
| 暴击率 | 玩家暴击率 |
| 暴击伤害 | 玩家暴击伤害倍率 |

> 如果数值公式过于复杂（比如需要同时引用多个角色或历史事件），应该用自定义效果脚本，不要试图在编辑器里堆字段。

## 6. 条件配置

每个效果可以配置零个或多个条件。只有条件满足时效果才会执行。

### 6.1 条件逻辑

当配置了多个条件时，可以选择：

| 逻辑 | 意思 |
|------|------|
| 全部满足 (AND) | 所有条件都满足才执行 |
| 任意满足 (OR) | 任一条件满足就执行 |

### 6.2 条件类型

| 类型 | 说明 | 需要填的字段 |
|------|------|-------------|
| 发起者属性检查 | 效果发起者的某属性达到指定值 | 属性、比较方式、比较值 |
| 目标属性检查 | 效果目标的某属性达到指定值 | 属性、比较方式、比较值 |
| 事件发生 | 某个特定事件是否发生过 | 事件名称 |
| 发起者状态检查 | 发起者是否处于某状态 | 状态类型 |
| 目标状态检查 | 目标是否处于某状态 | 状态类型 |
| 本回合计数 | 本回合内某计数达到指定值 | 属性、比较方式、比较值 |
| 本场战斗计数 | 本场战斗某计数达到指定值 | 属性、比较方式、比较值 |
| 自定义 | 绑定自定义条件脚本 | 脚本资源 |

### 6.3 比较方式

| 符号 | 意思 |
|------|------|
| < | 小于 |
| ≤ | 小于等于 |
| = | 等于 |
| ≥ | 大于等于 |
| > | 大于 |
| ≠ | 不等于 |

### 6.4 条件配置示例

**示例 1：若暴击则抽 1 张牌**

效果「抽 1 张牌」的条件：
- 条件类型 = 事件发生
- 事件名称 = `OnCrit`

**示例 2：若玩家本回合未受到伤害则获得 5 点护甲**

效果「获得 5 点护甲」的条件：
- 条件类型 = 本回合计数
- 属性 = （需通过自定义条件脚本或计数器实现）
- 比较方式 = =
- 比较值 = 0

**示例 3：若目标处于破甲状态则造成额外伤害**

效果「造成额外伤害」的条件：
- 条件类型 = 目标状态检查
- 状态类型 = 破甲

**示例 4：若本场累计失去理智达到 10 则抽 2 张牌**

效果「抽 2 张牌」的条件：
- 条件类型 = 本场战斗计数
- 属性 = 本场战斗累计失去理智
- 比较方式 = ≥
- 比较值 = 10

## 7. 自定义脚本

当编辑器的通用字段无法表达某种效果、条件或目标选择时，可以绑定自定义脚本。

> **策划不需要写代码**。程序员会写好脚本并创建为 `.asset` 资产，策划只需要在编辑器中拖入对应字段即可。

### 7.1 四类可绑定脚本

| 脚本类型 | 用途 | 拖入位置 |
|---------|------|---------|
| 自定义效果脚本 | 执行特殊效果（如增加热度、修改牌库结构等） | 效果类型选「自定义」后的「效果脚本」字段 |
| 自定义条件脚本 | 判断特殊条件（如热度大于 15） | 条件类型选「自定义」后的「条件脚本」字段 |
| 自定义卡牌脚本 | 处理整张卡的特殊流程（自定义目标选择、特殊结算顺序、玩家二次选择等） | 中栏的「自定义卡牌脚本」字段 |
| 自定义能力脚本 | 处理能力的特殊触发逻辑 | 能力编辑中的「自定义能力脚本」字段 |

### 7.2 什么时候用自定义脚本

**能用通用字段就用通用字段**，以下情况才考虑自定义脚本：

- 特殊目标选择（如「选择手牌中的一张牌弃掉」）
- 复杂数值公式（同时依赖多个角色或历史事件）
- 修改牌库结构（如创建、复制、替换卡牌）
- 非标准结算顺序
- 需要等待动画或玩家选择的异步流程
- 全新机制尚未被通用效果系统支持

### 7.3 策划操作步骤

1. 效果类型选择「自定义」
2. 将程序员提供的效果脚本 `.asset` 资源拖入「效果脚本」字段
3. 如果脚本需要参数，在「自定义参数」文本框中填写（具体格式问程序员）
4. 脚本名称会自动显示在下方

## 8. 品级配置

品级影响商店价格和刷新概率。**这些参数不在卡牌中配置**，而是在独立的品级配置窗口中统一调整。

### 8.1 打开品级配置

菜单 → `Tools` → `卡牌编辑器` → `品级配置`

### 8.2 可配置参数

每个品级可以配置：

| 参数 | 说明 |
|------|------|
| 商店基础价格 | 该品级卡牌在商店中的基础售价 |
| 商店刷新权重 | 商店刷新时该品级卡牌的出现概率权重 |
| 奖励刷新权重 | 战斗奖励中该品级卡牌的出现概率权重 |

### 8.3 默认值

| 品级 | 商店基础价格 | 商店刷新权重 | 奖励刷新权重 |
|------|------------|------------|------------|
| 铜 | 50 | 3 | 3 |
| 银 | 100 | 2 | 2 |
| 金 | 200 | 1 | 1 |

修改后点击「保存配置」即可。

## 9. 能力卡配置

当卡牌类型选择「能力」时，右栏会自动切换为能力编辑模式。

### 9.1 能力字段

| 字段 | 说明 |
|------|------|
| 能力名称 | 这个能力叫什么 |
| 能力描述 | 能力的效果说明 |
| 触发时机 | 什么时候自动触发（见下表） |
| 触发条件 | 满足什么条件才触发（可选，配置方式同效果条件） |
| 总触发次数 | 限制总共能触发几次（0 = 无限制） |
| 每回合触发次数 | 限制每回合能触发几次（0 = 无限制） |
| 触发后执行的效果 | 触发时执行哪些效果（就是普通的效果列表） |
| 自定义能力脚本 | 如果触发逻辑太复杂，可以绑定脚本 |

### 9.2 触发时机

| 时机 | 说明 |
|------|------|
| 回合开始 | 每个回合开始时 |
| 回合结束 | 每个回合结束时 |
| 暴击时 | 玩家打出暴击时 |
| 失去理智时 | 玩家失去理智时 |
| 理智低于阈值 | 玩家理智降到某个值以下时 |
| 获得减益时 | 玩家被施加减益时 |
| 每回合首次攻击 | 每回合第一次攻击时 |
| 施加破甲时 | 给敌人施加破甲时 |
| 自定义事件 | 需要绑定自定义能力脚本 |

### 9.3 能力配置示例

**节拍器**：每回合第一次攻击必定暴击

- 触发时机 = 每回合首次攻击
- 每回合触发次数 = 1
- 触发后效果 = 施加状态「暴击率提升」100 层，持续一次行动

## 10. 升级配置

### 10.1 基本规则

- 在中栏勾选「升级」后，才能配置升级版。
- 升级版有**独立**的费用、描述和效果列表。
- 升级效果不是基础效果的差值，而是一套完整的效果。

### 10.2 快捷操作

右栏顶部在升级模式下会显示两个按钮：

| 按钮 | 功能 |
|------|------|
| 从基础复制到升级 | 把当前基础效果复制一份到升级效果列表，方便在其基础上修改 |
| 清空升级效果 | 清空升级效果列表（基础效果不受影响） |

### 10.3 切换预览

右栏顶部有「基础效果」和「升级效果」两个标签页按钮，点击切换查看和编辑哪一版。

## 11. 卡牌操作

### 11.1 新建卡牌

左栏顶部「新建卡牌」按钮。会创建一张空白卡，自动选中并跳到中栏编辑。

### 11.2 复制卡牌

选中一张卡后点击「复制卡牌」按钮。会完整复制所有数据（包括效果列表），卡牌名称后缀加「(副本)」，ID 后缀加 `_copy`。

### 11.3 删除卡牌

选中一张卡后点击「删除卡牌」按钮。会弹出确认对话框，确认后删除 `.asset` 文件并从数据库移除。

### 11.4 搜索和筛选

左栏顶部搜索框可以按名称搜索。勾选「筛选」后可按品级、类型、词条和费用筛选。

## 12. 校验

### 12.1 单卡校验

在右栏底部「校验结果」区域，点击「校验当前卡牌」按钮。会检查以下内容：

| 检查项 | 级别 |
|-------|------|
| 卡牌 ID 为空 | 错误 |
| 卡牌 ID 重复 | 错误 |
| 卡牌名称为空 | 错误 |
| 费用小于 0 | 错误 |
| 攻击次数小于 1 | 错误 |
| 可升级卡缺少升级效果 | 警告 |
| 自定义效果未绑定脚本 | 错误 |
| 自定义条件未绑定脚本 | 错误 |
| 能力卡缺少触发效果 | 错误 |
| 持续指定回合但回合数为 0 | 错误 |
| 勾选施加破甲但数值为 0 | 警告 |
| 缺少卡面原画 | 警告 |
| 本局永久效果需确保有存档接口 | 提示 |

### 12.2 批量校验

顶部工具栏点击「批量验证」按钮。会校验数据库中所有卡牌，弹窗显示有错误的卡牌列表。

## 13. 测试卡牌

### 13.1 打开测试窗口

顶部工具栏点击「测试卡牌」按钮，或菜单 → `Tools` → `卡牌编辑器` → `测试窗口`。

### 13.2 配置测试参数

在测试窗口中可以配置模拟战斗的初始状态：

**玩家属性**：生命、最大生命、力量、敏捷、暴击率、暴击伤害、理智、能量、流血值

**敌人属性**：数量、生命、最大生命、护甲、流血值、破甲值

**随机设置**：随机种子（同样的种子结果可复现）

### 13.3 执行测试

点击「执行测试」按钮，下方会显示：

- 费用消耗
- 卡牌去往哪里（弃牌堆 / 移除）
- 每个效果的执行过程
- 条件是否满足
- 每段攻击的伤害和暴击判定
- 属性变化
- 护甲变化
- 能量变化
- 状态变化
- 能力触发情况

### 13.4 看懂测试日志

日志格式示例：

```
═══ 开始测试卡牌: 连环枪 (基础版) ═══
初始状态: 玩家HP=100 力量=0 敏捷=0 能量=3 暴击率=10%
初始状态: 敌人HP=50 护甲=0 破甲=0 流血=0

[费用] 消耗 2 点能量，剩余 1
[存在形式] 普通 — 使用后进入弃牌堆

--- 效果 1: 三连射 [伤害] ---
  [伤害] 基础伤害: 6 = 6
  第1击: 伤害=6 ✓暴击(x1.5) → 敌人HP=41/50
  第2击: 伤害=6 ✗未暴击 → 敌人HP=35/50
  第3击: 伤害=6 ✗未暴击 → 敌人HP=29/50

--- 效果 2: 暴击追加伤害 [伤害] ---
  条件: 事件[OnCrit]已发生 → ✓
  条件满足
  [伤害] 基础伤害: 3 = 3
  第1击: 伤害=3 ✗未暴击 → 敌人HP=26/50

═══ 测试结束 ═══
```

- `✓暴击` = 本次攻击暴击了
- `✗未暴击` = 本次攻击没暴击
- `条件: xxx → ✓/✗` = 条件是否满足
- 修改随机种子可以改变暴击判定结果

## 14. 完整示例：从零创建一张卡

以「穿甲弹」为例：

**目标**：费用 1 的攻击牌，造成 6 点伤害并施加 2 点破甲。升级版伤害提高到 9。

### 步骤

1. 点击「新建卡牌」
2. 中栏填写：
   - 卡牌 ID = `armor_piercing`
   - 卡牌名称 = `穿甲弹`
   - 品级 = 铜
   - 卡牌类型 = 攻击
   - 存在形式 = 普通
   - 基础费用 = 1
   - 勾选「升级」
   - 升级后费用 = 1
   - 基础描述 = `造成 6 点伤害并施加 2 点破甲。`
   - 升级后描述 = `造成 9 点伤害并施加 2 点破甲。`
3. 右栏（基础效果）：
   - 点击「+ 添加效果」
   - 效果名称 = `穿甲伤害`
   - 效果类型 = 伤害
   - 效果发起者 = 玩家
   - 效果目标 = 选定敌人
   - 公式类型 = 固定值
   - 基础数值 = 6
   - 攻击次数 = 1
   - 勾选「施加破甲」
   - 破甲数值 = 2
4. 切换到「升级效果」标签
5. 点击「从基础复制到升级」
6. 修改升级版效果的「基础数值」从 6 改为 9
7. 点击右栏底部「校验当前卡牌」确认无误
8. 顶部工具栏点击「测试卡牌」进行测试

完成！

## 15. 常见问题

### Q：效果执行顺序不对怎么办？

A：用效果右侧的 ↑ ↓ 按钮调整顺序。列表从上到下就是执行顺序。

### Q：我想让一个效果只在暴击时触发，怎么配？

A：给这个效果添加条件：条件类型选「事件发生」，事件名称填 `OnCrit`。

### Q：升级效果和基础效果可以完全不同吗？

A：可以。升级效果是一套独立完整的效果列表，数量、类型、目标都可以和基础版不同。

### Q：能力卡和普通卡有什么区别？

A：普通卡打出后立即执行效果然后进弃牌堆。能力卡打出后会一直挂在角色身上，等特定事件发生时自动触发效果。详见第 1.3 节和第 9 节。

### Q：护甲能不能配置成永久的？

A：不能在卡牌里配。护甲是否跨回合保留由玩家全局战斗属性决定（比如获得特定遗物或能力后）。

### Q：我想做一个「选择手牌中的一张牌弃掉并抽 2 张牌」的效果怎么办？

A：这种效果无法用通用字段表达。请找程序员写一个自定义效果脚本，然后拖入效果类型「自定义」的脚本字段。

### Q：「热度」是什么？编辑器里没有这个属性。

A：热度是游戏的自定义机制，不在通用属性列表中。引用热度的条件需要绑定自定义条件脚本。增加热度需要绑定自定义效果脚本。

### Q：生成的示例卡牌在哪里？

A：`Assets/ScriptableObjects/CardEditor/` 目录下。数据库在 `Assets/Resources/CardEditor/CardDatabase.asset`。

### Q：卡牌 ID 重复了怎么办？

A：编辑器校验会提示错误。修改其中一张卡的 ID 使其唯一即可。

### Q：批量验证报错了怎么看？

A：点击顶部「批量验证」按钮，会弹窗显示所有有错误的卡牌及其错误信息。
