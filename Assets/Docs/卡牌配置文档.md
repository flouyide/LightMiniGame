# 卡牌配置文档 & 新建教学

> 本文档面向策划人员，介绍如何在 Unity 中创建和配置新卡牌。

---

## 一、快速新建卡牌（5步）

### 第1步：选择对应类型的模板Prefab

在 Project 窗口中找到卡牌目录：

```
Assets/Prefabs/Battle/Cards/
```

目录下有3种**模板Prefab**：

| 模板Prefab名 | 类型 | 用途 |
|-------------|------|------|
| `攻击牌.prefab` | 攻击 | 制作AK47、棒球棒等攻击卡牌 |
| `护甲牌.prefab` | 护甲 | 制作防弹衣、迷彩服等护甲卡牌 |
| `增益牌.prefab` | 增益 | 制作力量提升、抽牌等增益卡牌 |

**根据你要做的卡牌类型，选中对应模板**，按 **Ctrl+D**（Mac: **Cmd+D**）复制一份。

> 卡牌类型由模板决定，复制后不可更改。Inspector 中 Card Type 字段为**只读**。

### 第2步：重命名

将复制出来的Prefab重命名为新卡牌的名字，例如 `手雷.prefab`。

### 第3步：在Inspector中配置属性

选中Prefab后，Inspector 中的 `Card Display` 组件**只会显示当前类型相关的配置项**：

**攻击牌模板** → 显示：基础信息 + 通用属性 + **攻击属性**

**护甲牌模板** → 显示：基础信息 + 通用属性 + **护甲属性**

**增益牌模板** → 显示：基础信息 + 通用属性 + **增益属性**

不相关的属性区域**不会出现**，无需关心。

### 第4步：填写卡牌属性

根据下方配置说明填写各字段。

### 第5步：加入牌组

打开 `Assets/Scenes/Battle.scene`，在 Hierarchy 中选中 `BattleSystem`，找到 `Battle Manager` → `Deck` 列表，点 `+` 将新卡牌Prefab拖入。

## 二、配置字段说明

### 2.1 基础信息（所有类型通用）

| 字段 | 说明 |
|------|------|
| **Card Type** | 只读，由模板决定（攻击/护甲/增益） |
| **Card Name** | 卡牌显示名称，如"AK47"、"防弹衣" |
| **Card Art** | 卡牌插图Sprite。不填则显示类型色半透明占位 |
| **Description** | 卡牌描述。**留空则自动生成**；填写后使用自定义文案 |

### 2.2 通用属性（所有类型通用）

| 字段 | 说明 |
|------|------|
| **Value** | 商店价值（金币） |
| **Grade** | 品级（见下表） |
| **Action Point Cost** | 打出消耗的行动点 |
| **Consume Type** | 消耗类型（见下表） |
| **Keywords** | 词条，可多选（见下表） |

#### 品级

| 值 | 中文 | 边框颜色 |
|----|------|----------|
| Common | 普通 | 灰 |
| Fine | 优秀 | 绿 |
| Rare | 精良 | 蓝 |
| Epic | 史诗 | 紫 |
| Legendary | 传说 | 金 |

#### 消耗类型

| 值 | 中文 | 效果 |
|----|------|------|
| None | 不消耗 | 进入弃牌堆，可再次抽到 |
| ThisBattle | 本战消耗 | 本场战斗永久移除 |
| ThisRun | 本局消耗 | 本局游戏永久移除 |

#### 词条

| 值 | 词条名 | 效果 | 适用 |
|----|--------|------|------|
| Pierce | 穿透 | 攻击无视护甲 | 攻击 |
| Lifesteal | 吸血 | 回复伤害的50%生命 | 攻击 |
| Combo | 连击 | 攻击次数+1 | 攻击 |
| Heavy | 重击 | 25%概率双倍伤害 | 攻击 |
| Swift | 迅捷 | 打出后抽1张牌 | 攻击 |
| Toxic | 剧毒 | 额外2点伤害（无视护甲） | 攻击 |
| Burning | 燃烧 | 额外3点伤害（无视护甲） | 攻击 |
| Sturdy | 坚固 | 护甲不随回合消失 | 护甲 |
| FreePlay | 免费 | 不消耗行动点 | 全部 |

---

### 2.3 攻击属性（仅攻击牌模板显示）

| 字段 | 说明 |
|------|------|
| **Attack Count** | 攻击次数 |
| **Attack Value Type** | `Fixed`(固定值) 或 `AttributeBased`(属性加成) |
| **Attack Value** | 基础攻击数值 |
| **Attack Attribute** | 当Value Type=AttributeBased时，附加的玩家属性（仅此模式显示） |
| **Ignore Armor** | 是否无视敌人护甲 |

#### 攻击数值示例

| 配置 | 实际效果 |
|------|----------|
| Fixed, Attack Value = 6 | 固定6点 |
| AttributeBased, Attack Value = 4, Strength | 4 + 玩家力量 |
| Attack Count = 3, Attack Value = 3 | 3×3 = 共9点 |
| Keywords勾选Combo, Attack Count = 2 | 实际打3次 |

---

### 2.4 护甲属性（仅护甲牌模板显示）

| 字段 | 说明 |
|------|------|
| **Armor Value Type** | `Fixed`(固定值) 或 `AttributeBased`(属性加成) |
| **Armor Value** | 基础护甲值 |
| **Armor Attribute** | 当Value Type=AttributeBased时，附加的玩家属性（仅此模式显示） |

#### 护甲数值示例

| 配置 | 实际效果 |
|------|----------|
| Fixed, Armor Value = 6 | 固定6点护甲 |
| AttributeBased, Armor Value = 4, Dexterity | 4 + 玩家敏捷 |

> 默认护甲回合结束时清零，勾选Sturdy词条则保留。

---

### 2.5 增益属性（仅增益牌模板显示）

| 字段 | 说明 |
|------|------|
| **Buff Duration** | 增益时效（见下表） |
| **Buff Duration Turns** | 回合数（仅Duration=BattleXTurns时显示） |
| **Buff Stacks** | 增益层数，效果 = 单个效果值 × 层数 |
| **Buff Effects** | 增益效果列表，可添加多个 |

#### 增益时效

| 值 | 中文 | 效果 |
|----|------|------|
| GlobalPermanent | 全局永久 | 跨战斗保留 |
| BattlePermanent | 局内永久 | 本场战斗内生效 |
| BattleXTurns | 局内x回合 | 生效指定回合数 |

#### 增益效果

在Buff Effects列表点 `+` 添加，每个效果：

| 字段 | 说明 |
|------|------|
| **Effect Type** | 效果类型（见下表） |
| **Value** | 效果数值 |
| **Target Attribute** | 目标属性（仅Effect Type=IncreaseAttribute时显示） |

| 值 | 中文 | 效果 |
|----|------|------|
| IncreaseAttribute | 提升属性 | 永久提升指定属性 |
| RestoreActionPoints | 回复行动力 | 立即回复行动点 |
| DrawCards | 抽牌 | 立即抽牌 |
| GainArmor | 获得护甲 | 立即获得护甲 |
| HealHP | 回复生命 | 立即回复生命 |

#### 玩家属性

| 值 | 中文 | 影响 |
|----|------|------|
| Strength | 力量 | 攻击牌属性加成 |
| Dexterity | 敏捷 | 护甲牌属性加成 |
| Vitality | 体质 | 最大生命值 |
| Agility | 灵巧 | 每回合抽牌数 |

## 三、完整配置示例

### 示例1：AK47（攻击牌）

```
模板来源: 攻击牌.prefab
Card Name:    AK47
Card Art:     (拖入插图)
Value:        35
Grade:        Fine
Action Point: 2
Consume Type: None
Keywords:     Pierce

攻击属性:
  Attack Count:      1
  Attack Value Type: Fixed
  Attack Value:      10
  Ignore Armor:      false
```

### 示例2：防弹衣（护甲牌）

```
模板来源: 护甲牌.prefab
Card Name:    防弹衣
Value:        30
Grade:        Fine
Action Point: 1
Consume Type: None
Keywords:     Sturdy

护甲属性:
  Armor Value Type: Fixed
  Armor Value:      6
```

### 示例3：力量提升（增益牌）

```
模板来源: 增益牌.prefab
Card Name:    力量提升
Value:        28
Grade:        Fine
Action Point: 1
Consume Type: None
Keywords:     None

增益属性:
  Buff Duration:  BattlePermanent
  Buff Stacks:    1
  Buff Effects:
    → Effect Type:      IncreaseAttribute
    → Value:            2
    → Target Attribute: Strength
```

## 四、注意事项

### ⚠️ 不要修改的字段

Inspector中 `Card Display` 组件下方的 `UI引用`、`类型颜色`、`品级颜色`、`不可打出状态` 区域为内部引用，**不要手动拖拽或清空**。

### ⚠️ Description 留空 vs 自定义

- **留空**：系统根据数值参数自动生成描述，改数值时描述自动更新
- **自定义**：填写后使用固定文案，改数值不影响显示

### ⚠️ 词条与类型匹配

配错不会报错但无效果：
- `Sturdy` 只对护甲牌有效
- `Pierce`/`Combo`/`Heavy`/`Toxic`/`Burning`/`Lifesteal` 只对攻击牌有效
- `Swift`/`FreePlay` 对所有类型有效

### ⚠️ 加卡到牌组

配置完后需加入战斗牌组：打开 Battle.scene → 选中 BattleSystem → Deck 列表 → 拖入Prefab
