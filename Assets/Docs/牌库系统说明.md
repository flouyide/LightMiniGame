# 牌库系统 · 策划配置指南

> 给**策划**看的手册：不碰代码，只讲「在编辑器里点什么、填什么、怎么验证」。
> 相关：遗物配置见 `遗物系统说明.md`，商店卖哪些卡见 `商店系统说明.md`。

---

## 0. 一句话理解牌库

「牌库」就是**每个角色各自拥有的卡牌集合**。角色 A 的牌和角色 B 的牌是分开的，互不干扰。
游戏里一张能用的卡 = **一张「卡牌模板」（配置好的定义，全游戏共用）** + **一份「属于某角色的实例」（玩家真正拥有的那张）**。

- 同一张模板可以存在于多个角色牌库里（甚至同一角色拥有多张同模板，每张独立）。
- 玩家买卡 / 删卡，都只影响「某个角色」的实例，不会动到模板。

## 1. 策划配置总清单

按这个顺序做，缺一步后面都可能跑不通：

- [ ] **① 建卡牌模板**（`CardData`）——定义每张卡是什么
- [ ] **② 建角色**（`CharacterData`）——定义角色身份/头像/容量
- [ ] **③ 建初始牌组**（`CharacterStartingLibrary`）——开局给角色发哪些卡
- [ ] **④ 串联挂接**——把上面资产拖到正确位置
- [ ] **⑤ 卡面预制体**——让牌库界面/商店能画出卡牌外观

---

## 2. ① 建卡牌模板（CardData）

**操作**：Project 窗口右键 → `Create → CardGame → Card Data`，重命名为 `卡_xxx`。

双击打开后在 Inspector 填：

| 字段 | 策划要填什么 | 说明 / 例子 |
| --- | --- | --- |
| `cardName` | 卡牌显示名 | 「火球术」 |
| `cardType` | 卡牌类型 | 下拉选 `Attack`(攻击) / `Armor`(护甲) / `Buff`(增益)。**这决定用哪张卡面预制体画出来**，务必选对 |
| `cardArt` | 卡牌插画（Sprite） | 拖一张图；不填则卡面无图 |
| `description` | 描述文本 | 卡牌效果说明，会显示在卡面上 |
| `value` | 商店售价（金币） | 如 `40`。商店卖这张卡时标这个价 |
| `grade` | 品级 | 下拉选品级，用于排序/显示「稀有度」 |
| `actionPointCost` | 出牌消耗的行动点 | 战斗数值，按设计填 |
| `consumeType` | 消耗类型 | 一次性 / 持续等，按设计填 |
| `keywords` | 关键词标签 | 可选，用于检索/显示 |
| `attack* / armor* / buff*` | 具体数值 | 按卡牌类型填对应数值（攻击卡填攻击相关，等） |

> 静态字段（程序自动算）：品级名、类型名由程序读取，不用手填。
> **一句话**：`cardName / cardType / value` 是最关键的三个；其余按你的数值设计填。

---

## 3. ② 建角色（CharacterData）

**操作**：Project 右键 → `Create → CardGame → Character Data`，重命名为角色名。

| 字段 | 策划要填什么 | 说明 |
| --- | --- | --- |
| `characterId` | 英文/数字稳定 ID | **不能重复、不能随便改**（它是牌库/遗物库的「钥匙」）。例：`hero1` |
| `displayName` | 角色显示名 | 「女剑士」，显示在牌库/遗物标题和按钮上 |
| `avatar` | 角色头像（Sprite） | 牌库/遗物界面的角色切换按钮会显示这个头像 |
| `startingLibrary` | 初始牌组资产 | 见步骤 ③，先建好再拖回来 |
| `startingRelics` | 初始遗物 | 见 `遗物系统说明.md` 步骤 ④ |
| `maxLibrarySize` | 牌库容量上限 | **≤0 = 不限制**；填正数则牌库满后不能再买卡（商店返回「牌库已满」） |

> ⚠️ `characterId` 一旦被存档/牌库引用，**不要改**；要改名改 `displayName` 即可。

---

## 4. ③ 建初始牌组（CharacterStartingLibrary）

**操作**：Project 右键 → `Create → CardGame → Character Starting Library`，重命名如 `起始牌组_女剑士`。

| 字段 | 策划要填什么 | 说明 |
| --- | --- | --- |
| `character` | 拖入对应 `CharacterData` | 这个牌组属于谁 |
| `startingCards` | 拖入卡牌模板列表 | 开局发给该角色的卡。**同一张卡拖多次 = 开局就有多张**（每张独立） |

填好后，把这个资产拖回 `CharacterData.startingLibrary`（步骤 ② 那个字段）。

---

## 5. ④ 串联挂接（最关键的一步）

| 要做的事 | 在哪拖 | 拖什么 |
| --- | --- | --- |
| 角色进游戏 | `GameConfig.asset → characters` 列表 | 把所有 `CharacterData` 资产逐个拖进去（顺序 = 角色1、角色2…） |
| 初始牌组生效 | `CharacterData.startingLibrary` | 拖入对应的 `CharacterStartingLibrary` |

> 顺序很重要：列表里**第 0 个 = 角色1，第 1 个 = 角色2**。商店的「角色比例」「清单界面按钮」都按这个顺序对应。

---

## 6. ⑤ 卡面预制体（让卡牌画得出来）

牌库界面和商店里的卡牌，靠三张「卡面预制体」按 `cardType` 画出来：

- `Assets/Prefabs/Battle/Cards/攻击牌.prefab`
- `Assets/Prefabs/Battle/Cards/护甲牌.prefab`
- `Assets/Prefabs/Battle/Cards/增益牌.prefab`

**挂接位置**：场景里 `CardLibraryPanel` 上的 `CardLibraryPanelUI` 组件，把三张预制体分别拖到
`attackCardPrefab / armorCardPrefab / buffCardPrefab`。

> 这三张预制体根节点必须挂 `CardDisplay` 组件，否则卡面画不出来。一般项目已备好，无需新建。

---

## 7. 常见坑 & 排错

| 现象 | 原因 / 解决办法 |
| --- | --- |
| 进游戏牌库是空的 | 没建 `CharacterStartingLibrary` 或没拖到 `CharacterData.startingLibrary`；或 `GameConfig.characters` 没放角色 |
| 卡牌显示「[缺少 X 预制体]」 | `CardLibraryPanelUI` 的 `攻击牌/护甲牌/增益牌` 预制体没拖（需对应 `cardType`） |
| 买了卡提示「牌库已满」 | 该角色 `maxLibrarySize` 设了正数且已达上限；调大或设 0（不限制） |
| 卡面没图 | `CardData.cardArt` 没拖插画 |
| 改了 `characterId` 后旧存档牌库错乱 | `characterId` 是钥匙，别改；改名用 `displayName` |

## 8. 验证清单（进 Unity 后做一遍）

- [ ] 游戏启动后打开牌库界面，能看到每个角色的初始卡，且角色切换按钮正常。
- [ ] 角色 A 删一张卡、角色 B 的牌库不受影响。
- [ ] 商店买卡后，卡进入**对应角色**牌库（看 `ownerCharacter`）。
- [ ] 卡牌外观由 `cardType` 正确选择预制体绘制。

> 项目使用定制引擎，无法命令行编译；任何配置改动后请在 Unity 打开工程确认无报错、并实机走一遍上述流程。
