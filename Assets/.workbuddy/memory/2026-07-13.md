## 修复 TextMeshPro 中文字符显示为方框（口）的问题

**现象：** 运行时报错 `The character with Unicode value \u91CD was not found in the [LiberationSans SDF] font asset or any potential fallbacks. It was replaced by Unicode character \u25A1`，所有中文显示成方框。

**根因：** `Assets/TextMesh Pro/Resources/TMP Settings.asset` 中 `m_fallbackFontAssets` 为空列表 `[]`。默认字体 LiberationSans SDF 只含拉丁字符，遇到中文（如"重"\u91CD）时无 fallback 可回退，于是显示为方框。

**修复：** 项目已有 `Assets/Fonts/SimHei SDF.asset`（动态图集，`m_AtlasPopulationMode: 1`，源字体 simhei.ttf），但未挂载。将其 GUID（`XS4dtiykBinVXiLiSwJudGfzlE2v78DFd87hxM3phGHPqsvPBQN82j8=`）加入 TMP Settings 的 `m_fallbackFontAssets` 列表。

**注意：** 团结引擎（Tuanjie）使用 Base64 格式 GUID，与标准 Unity 的 32 位十六进制 GUID 不同，但用法一致。

### 补充：发现并修复了更深层的根因（GUID 引用断裂）

仅配 fallback 后中文仍可能显示方框，因为 `SimHei SDF.asset` 内部对源字体 simhei.ttf 的 GUID 引用是断裂的：

- `SimHei SDF.asset` 内部三处引用（`m_SourceFontFileGUID`、`m_SourceFontFile_EditorRef`、`m_SourceFontFile`）用的是旧的标准 hex GUID `b0ea87d9b2148eb479a9f62fae8855bb`
- 但 `simhei.ttf.meta` 实际 GUID 是团结引擎 Base64 格式 `XXpK5CKqB3IhSgCpiqAwuNdVKvRmblDbFz8f0dRv2ZAJ0hB4dlmxI5U=`
- 二者不匹配 → SimHei SDF 找不到源字体 → 动态图集无法渲染中文字符 → 即使配了 fallback 也无效

**修复：** 将 `SimHei SDF.asset` 内三处 `b0ea87d9b2148eb479a9f62fae8855bb` 全部替换为 `XXpK5CKqB3IhSgCpiqAwuNdVKvRmblDbFz8f0dRv2ZAJ0hB4dlmxI5U=`。

**经验：** 团结引擎项目从标准 Unity 迁移资源时，.meta 文件可能被重新生成为新 Base64 GUID，但 .asset 文件内部对其它资源的 GUID 引用不会自动更新，导致引用断裂。排查此类问题应对比 .asset 内部 guid 引用与目标资源 .meta 的 guid 是否一致。引擎内两种 GUID 格式并存（老资源 hex，新资源 Base64），跨格式引用需手动修正。

## 修正：.asset 文件内部不支持 Base64 GUID（之前的 GUID 修复有误）

**问题：** 之前把 `SimHei SDF.asset` 内三处源字体 GUID 和 `TMP Settings.asset` 的 fallback 引用手动改成了 Base64 格式（simhei.ttf.meta 的 GUID），导致团结引擎报错 `Could not extract GUID in text file ... at line 20`。

**根因：** 团结引擎的 `.asset` 文件内部 `guid:` 字段**只支持标准 32 位 hex 格式**（如 `b0ea87d9b2148eb479a9f62fae8855bb`）。Base64 格式（含 `+`/`/`/`=` 字符）是 `.meta` 文件专用的，引擎内部有 Base64→hex 映射机制，但这个映射只在**编辑器 Inspector 里拖入引用时自动处理**，不能手动写入 .asset 文件。

**修正：** 已将两个文件还原——SimHei SDF.asset 三处 GUID 改回 hex `b0ea87d9b2148eb479a9f62fae8855bb`（Missing 引用），TMP Settings fallback 改回空列表 `[]`。

**正确操作（需在编辑器里做）：**
1. 选中 SimHei SDF.asset → Inspector → Source Font File → 拖入 simhei.ttf
2. 选中 TMP Settings.asset → Inspector → Fallback Font Assets → 点 + → 拖入 SimHei SDF

**重要经验：** 团结引擎项目中，引用 .meta GUID 为 Base64 格式的资源时，**不能手动编辑 .asset 文件内的 guid 字段**，必须通过编辑器 Inspector 操作让引擎自动写入正确的 hex GUID。
